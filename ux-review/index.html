Overall Application Issues

These issues affect the entire application and should be addressed holistically:

I1 No Persistent Navigation Critical

Problem: The application has no navigation menu. Users can't easily move between sections (Search, TOC List, Dashboard). The only navigation is the logo (links to home) and contextual "Back" links.

Impact: Users get lost, can't discover features, and rely on browser back button.

Recommendation R1: High Impact Add a persistent navigation bar with: Home, Search Books, TOC List, My Contributions, Help

I2 Workflow is Invisible Critical

Problem: The five-step workflow (Search â†’ Create â†’ Browse Scans â†’ Transcribe â†’ Verify) is never explained. Users don't understand the big picture or what state their work is in.

Impact: New users are confused about what to do next. No sense of progress or accomplishment.

Recommendation R2: High Impact Add workflow visualization showing current step, completed steps, and next steps. Include on TOC edit/show pages.

I3 Visual Design is Minimal High

Problem: The UI uses basic Bootstrap 3 with almost no customization. No color scheme, no visual identity beyond logo, default typography everywhere.

Impact: Application feels outdated and unprofessional. Hard to establish brand trust.

Recommendation R3: Medium Impact Implement a design system with: custom color palette, consistent typography scale, button styles, card components, proper spacing system.

I4 No User Dashboard High

Problem: Signed-in users land on homepage with two buttons. No overview of their contributions, pending verifications, or project statistics.

Impact: Users don't know what to work on or see their impact. No sense of community or progress.

Recommendation R4: High Impact Create a dashboard showing: my recent TOCs, TOCs needing verification, project stats, activity feed.

I5 Mobile Experience is Broken Critical

Problem: Wide tables (TOC index), two-column layouts (edit form), and no mobile-specific considerations make the app unusable on phones.

Impact: ~50% of potential users can't use the app effectively.

Recommendation R5: High Impact Implement mobile-first responsive design: cards instead of tables, collapsible sections, mobile-optimized forms.

I6 Inconsistent Spacing and Alignment Medium

Problem: Inline styles scattered throughout views create inconsistent spacing. Some sections use `style: 'margin-top: 50px;'`, others use Bootstrap utility classes.

Impact: Visual inconsistency, harder to maintain, unprofessional appearance.

Recommendation R6: Medium Impact Establish a spacing system (4px base unit) and remove all inline styles. Use utility classes consistently.

I7 No Contextual Help High

Problem: Complex features (OCR, subject headings, markdown format) have minimal explanation. Users must figure it out themselves.

Impact: High learning curve, errors, frustration, abandonment.

Recommendation R7: High Impact Add tooltips, inline help text, and a help sidebar with examples for each screen.

I8 No Error Prevention Medium

Problem: Users can create TOCs without understanding the workflow, submit empty forms, or verify their own work (currently blocked but not explained upfront).

Impact: Wasted effort, confusion, need to start over.

Recommendation R8: Medium Impact Add progressive disclosure: show only relevant fields/actions at each step. Disable unavailable actions with explanations.
